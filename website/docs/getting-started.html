Getting Started with R3E DevPack

Welcome to the R3E Neo Contract DevPack! This guide will help you get up and running with developing smart contracts for the Neo blockchain using our comprehensive toolkit.

Installation

The R3E Neo Contract Compiler (RNCC) is available as a self-contained binary for all major platforms. Choose your platform and follow the installation instructions:

Windows Installation

Download and run the Windows executable:

# Download using PowerShell
Invoke-WebRequest -Uri "https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-win-x64.exe" -OutFile "rncc.exe"

# Verify installation
.\rncc.exe --version

# Add to PATH (optional, requires admin)
$path = [Environment]::GetEnvironmentVariable("PATH", "User")
[Environment]::SetEnvironmentVariable("PATH", "$path;$PWD", "User")

Linux Installation

Download and install on Linux:

# Download the binary
wget https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-linux-x64

# Make it executable
chmod +x rncc-linux-x64

# Move to system path (optional)
sudo mv rncc-linux-x64 /usr/local/bin/rncc

# Verify installation
rncc --version

macOS Installation

For Intel Macs:

# Download for Intel
curl -L https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-macos-x64 -o rncc
chmod +x rncc
sudo mv rncc /usr/local/bin/

# Verify
rncc --version

For Apple Silicon (M1/M2):

# Download for Apple Silicon
curl -L https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-macos-arm64 -o rncc
chmod +x rncc
sudo mv rncc /usr/local/bin/

# Verify
rncc --version

Your First Smart Contract

Let's create a simple "Hello World" smart contract to get familiar with the development process.

1. Create the Contract File

Create a new file called HelloWorld.cs:

using Neo.SmartContract.Framework;
using Neo.SmartContract.Framework.Attributes;
using Neo.SmartContract.Framework.Native;
using Neo.SmartContract.Framework.Services;

namespace HelloWorld
{
    [DisplayName("HelloWorld")]
    [ManifestExtra("Author", "Your Name")]
    [ManifestExtra("Email", "your.email@example.com")]
    [ManifestExtra("Description", "A simple Hello World contract")]
    public class HelloWorld : SmartContract
    {
        // Contract owner
        [InitialValue("NiHYtfX8SdAQQJYf4fVvfcvfQQddH8WMLH", ContractParameterType.Hash160)]
        private static readonly UInt160 Owner = default;

        // Simple greeting method
        public static string SayHello(string name)
        {
            return $"Hello, {name}! Welcome to Neo blockchain.";
        }

        // Get contract owner
        public static UInt160 GetOwner()
        {
            return Owner;
        }

        // Update contract (only owner can do this)
        public static bool Update(ByteString nefFile, string manifest)
        {
            if (!IsOwner()) return false;
            ContractManagement.Update(nefFile, manifest);
            return true;
        }

        // Check if caller is owner
        private static bool IsOwner()
        {
            return Runtime.CheckWitness(Owner);
        }
    }
}

2. Understanding the Code

Key Concepts

  • SmartContract: Base class for all Neo smart contracts
  • Attributes: Metadata about your contract (name, author, etc.)
  • Methods: Public static methods become contract endpoints
  • Owner: Common pattern for contract administration
  • CheckWitness: Verifies caller's identity

Compiling Your Contract

Now let's compile the contract using RNCC:

Basic Compilation

# Compile the contract
rncc HelloWorld.cs

# This generates:
# - bin/sc/HelloWorld.nef (compiled contract)
# - bin/sc/HelloWorld.manifest.json (contract metadata)

Advanced Compilation Options

# Compile with optimization
rncc HelloWorld.cs --optimize=All

# Compile with debug information
rncc HelloWorld.cs -d Extended

# Generate all artifacts
rncc HelloWorld.cs --generate-artifacts=All --security-analysis

# Generate WebGUI
rncc HelloWorld.cs --generate-webgui

# Custom output directory
rncc HelloWorld.cs -o ./build

Pro Tip: Production Build

For production deployments, use this command for maximum optimization and security:

rncc HelloWorld.cs \
  --optimize=All \
  --security-analysis \
  --generate-artifacts=All \
  -o ./dist

Deployment

After compilation, you're ready to deploy your contract to the Neo blockchain.

1. Testing Locally

First, test your contract using Neo Express (local blockchain):

# Install Neo Express (if not already installed)
dotnet tool install Neo.Express -g

# Create a local blockchain
neoxp create

# Deploy contract
neoxp contract deploy HelloWorld.nef genesis

2. TestNet Deployment

Deploy to Neo TestNet for public testing:

# Using neo-cli
neo-cli
neo> open wallet your-wallet.json
neo> deploy HelloWorld.nef HelloWorld.manifest.json

3. WebGUI Deployment

If you generated a WebGUI, deploy it to R3E hosting:

# Generate and deploy WebGUI
rncc HelloWorld.cs \
  --generate-webgui \
  --deploy-webgui \
  --contract-address=0x123... \
  --network=testnet \
  --deployer-address=NiHYtfX...
