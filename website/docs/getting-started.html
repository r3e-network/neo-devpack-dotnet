Installation

The R3E Neo Contract Compiler (RNCC) is available as a self-contained binary for all major platforms. Choose your platform and follow the installation instructions:

Windows Installation

Download and run the Windows executable:

# Download using PowerShell
Invoke-WebRequest -Uri "https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-win-x64.exe" -OutFile "rncc.exe"

# Verify installation
.\rncc.exe --version

# Add to PATH (optional, requires admin)
$path = [Environment]::GetEnvironmentVariable("PATH", "User")
[Environment]::SetEnvironmentVariable("PATH", "$path;$PWD", "User")

Linux Installation

Download and install on Linux:

# Download the binary
wget https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-linux-x64

# Make it executable
chmod +x rncc-linux-x64

# Move to system path (optional)
sudo mv rncc-linux-x64 /usr/local/bin/rncc

# Verify installation
rncc --version

macOS Installation

For Intel Macs:

# Download for Intel
curl -L https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-macos-x64 -o rncc
chmod +x rncc
sudo mv rncc /usr/local/bin/

# Verify
rncc --version

For Apple Silicon (M1/M2):

# Download for Apple Silicon
curl -L https://github.com/r3e-network/r3e-devpack-dotnet/releases/latest/download/rncc-macos-arm64 -o rncc
chmod +x rncc
sudo mv rncc /usr/local/bin/

# Verify
rncc --version

Creating Projects from Templates

R3E DevPack provides project templates to quickly scaffold new smart contract projects with proper structure and dependencies.

Using RNCC Templates

Create a new project using the built-in templates:

# Create a basic contract project
rncc new MyProject --template=basic

# Create a token contract project (NEP-17)
rncc new MyToken --template=nep17

# Create an NFT contract project (NEP-11)
rncc new MyNFT --template=nep11

# Create a DeFi contract project
rncc new MyDeFi --template=defi

# Create a full solution with tests
rncc new MySolution --template=solution --with-tests

Available Templates

Basic Contract

--template=basic

Simple smart contract with owner pattern and basic methods

NEP-17 Token

--template=nep17

Fungible token contract with standard NEP-17 implementation

NEP-11 NFT

--template=nep11

Non-fungible token contract with metadata support

DeFi Protocol

--template=defi

DeFi contract with staking, rewards, and governance

Multi-Sig Wallet

--template=multisig

Multi-signature wallet with proposal system

Full Solution

--template=solution

Complete solution with contracts, tests, and deployment scripts

Template Options

# Create with additional options
rncc new MyProject --template=nep17 \
  --author="Your Name" \
  --email="your@email.com" \
  --namespace="MyCompany.Contracts" \
  --with-tests \
  --with-deploy-scripts \
  --git-init

Quick Start Tip

Use --template=solution for a complete project setup including:

  • Contract implementation
  • Unit tests
  • Integration tests
  • Deployment scripts
  • Documentation
  • GitHub Actions workflow

Key Files and Directories

src/

Contains all contract source code organized by functionality:

  • Main contract files: Core contract logic
  • Storage files: Storage layout and access patterns
  • Event files: Event definitions and emitters
  • Helper files: Utility functions and common logic

tests/

Comprehensive test suite:

  • Unit tests: Test individual methods
  • Integration tests: Test contract interactions
  • Test utilities: Shared test helpers

deploy/

Deployment infrastructure:

  • Scripts: Automated deployment scripts
  • Config: Network-specific configurations
  • Artifacts: Compiled contracts and manifests

Configuration Files

  • .csproj: Project configuration with Neo.SmartContract.Framework reference
  • nuget.config: Package sources including Neo packages
  • Directory.Build.props: Shared build properties across projects
  • .editorconfig: Code style and formatting rules

Project File Configuration

A typical contract project file (.csproj) looks like this:

<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net7.0</TargetFramework>
    <Nullable>enable</Nullable>
    <RootNamespace>MyProject</RootNamespace>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Neo.SmartContract.Framework" Version="3.6.2" />
  </ItemGroup>

  <Target Name="PostBuild" AfterTargets="PostBuildEvent">
    <Exec Command="rncc $(TargetPath) -o $(TargetDir)" />
  </Target>
</Project>

Important Notes

  • Always use the latest version of Neo.SmartContract.Framework
  • The post-build event automatically compiles your contract after building
  • Keep sensitive information (private keys, passwords) out of source control
  • Use .gitignore to exclude build artifacts and sensitive files

Your First Smart Contract

Let's create your first smart contract project using R3E DevPack templates. This approach ensures you start with a properly structured solution.

1. Generate a New Contract Project

Use RNCC to create a new smart contract solution from a template:

# Create a basic contract project
rncc new HelloWorld --template=basic \
  --author="Your Name" \
  --email="your@email.com" \
  --with-tests

# Navigate to the project directory
cd HelloWorld

# View the generated structure
ls -la

What's Generated

The template creates a complete project structure with:

  • src/HelloWorld.cs - Main contract file with basic functionality
  • tests/HelloWorld.Tests.cs - Unit tests for your contract
  • HelloWorld.csproj - Project configuration with Neo dependencies
  • README.md - Project documentation template
  • .gitignore - Git configuration for Neo projects

2. Explore the Generated Contract

The template generates a well-structured contract. Let's look at the main contract file:

using Neo.SmartContract.Framework;
using Neo.SmartContract.Framework.Attributes;
using Neo.SmartContract.Framework.Native;
using Neo.SmartContract.Framework.Services;

namespace HelloWorld
{
    [DisplayName("HelloWorld")]
    [ManifestExtra("Author", "Your Name")]
    [ManifestExtra("Email", "your@email.com")]
    [ManifestExtra("Description", "A basic smart contract generated from R3E DevPack template")]
    [ContractSourceCode("https://github.com/yourusername/HelloWorld")]
    public class HelloWorld : SmartContract
    {
        // Contract owner - replace with your address
        [InitialValue("NiHYtfX8SdAQQJYf4fVvfcvfQQddH8WMLH", ContractParameterType.Hash160)]
        private static readonly UInt160 Owner = default;

        // Storage prefix keys
        private const byte Prefix_Config = 0x01;
        private const byte Prefix_Data = 0x02;

        // Events
        [DisplayName("Greeting")]
        public static event Action OnGreeting;

        // Deploy method - called once during deployment
        public static void _deploy(object data, bool update)
        {
            if (!update)
            {
                // Initialize contract during first deployment
                Storage.Put(new StorageContext(), new byte[] { Prefix_Config, 0x01 }, "initialized");
            }
        }

        // Main contract method
        public static string SayHello(string name)
        {
            // Validate input
            if (name.Length == 0 || name.Length > 100)
                throw new Exception("Invalid name length");

            // Get caller address
            var caller = Runtime.CallingScriptHash;
            
            // Fire event
            OnGreeting(caller, name);
            
            return $"Hello, {name}! Welcome to Neo blockchain.";
        }

        // Store a message (demonstrates storage usage)
        public static void StoreMessage(string key, string message)
        {
            if (!IsOwner()) throw new Exception("Only owner can store messages");
            
            var storageKey = new byte[] { Prefix_Data }.Concat(key.ToByteArray());
            Storage.Put(new StorageContext(), storageKey, message);
        }

        // Retrieve a message
        public static string GetMessage(string key)
        {
            var storageKey = new byte[] { Prefix_Data }.Concat(key.ToByteArray());
            return Storage.Get(new StorageContext(), storageKey);
        }

        // Contract management methods
        public static UInt160 GetOwner() => Owner;

        public static bool Update(ByteString nefFile, string manifest)
        {
            if (!IsOwner()) return false;
            ContractManagement.Update(nefFile, manifest);
            return true;
        }

        public static bool Destroy()
        {
            if (!IsOwner()) return false;
            ContractManagement.Destroy();
            return true;
        }

        // Helper methods
        private static bool IsOwner()
        {
            return Runtime.CheckWitness(Owner);
        }
    }
}

3. Understanding the Generated Code

Template Features

The generated contract includes best practices:

  • Owner Pattern: Administrative control for contract management
  • Storage Organization: Prefix-based storage key management
  • Event System: Events for important contract actions
  • Deploy Hook: Initialization logic during deployment
  • Input Validation: Proper validation of method parameters
  • Contract Lifecycle: Update and destroy capabilities

4. Review the Generated Tests

The template also creates unit tests for your contract:

using Neo.SmartContract.Testing;
using Neo.SmartContract.Testing.TestingStandards;
using Neo.VM;
using Xunit;

namespace HelloWorld.Tests
{
    public class HelloWorldTests : TestBase
    {
        [Fact]
        public void Test_SayHello_ValidInput()
        {
            // Arrange
            var snapshot = TestEngine.Snapshot;
            var contract = TestEngine.Deploy(snapshot, ContractPath);
            
            // Act
            var result = contract.SayHello("Alice");
            
            // Assert
            Assert.Equal("Hello, Alice! Welcome to Neo blockchain.", result);
        }

        [Fact]
        public void Test_SayHello_EmptyName_ThrowsException()
        {
            // Arrange
            var snapshot = TestEngine.Snapshot;
            var contract = TestEngine.Deploy(snapshot, ContractPath);
            
            // Act & Assert
            Assert.Throws(() => contract.SayHello(""));
        }

        [Fact]
        public void Test_StoreMessage_AsOwner_Success()
        {
            // Arrange
            var snapshot = TestEngine.Snapshot;
            var contract = TestEngine.Deploy(snapshot, ContractPath);
            TestEngine.SetSigner(Owner);
            
            // Act
            contract.StoreMessage("test-key", "Hello, World!");
            var retrieved = contract.GetMessage("test-key");
            
            // Assert
            Assert.Equal("Hello, World!", retrieved);
        }
    }
}

Compiling Your Contract

Now let's compile the contract using RNCC:

Basic Compilation

# Compile the contract
rncc HelloWorld.cs

# This generates:
# - bin/sc/HelloWorld.nef (compiled contract)
# - bin/sc/HelloWorld.manifest.json (contract metadata)

Advanced Compilation Options

# Compile with optimization
rncc HelloWorld.cs --optimize=All

# Compile with debug information
rncc HelloWorld.cs -d Extended

# Generate all artifacts
rncc HelloWorld.cs --generate-artifacts=All --security-analysis

# Generate WebGUI
rncc HelloWorld.cs --generate-webgui

# Custom output directory
rncc HelloWorld.cs -o ./build

Pro Tip: Production Build

For production deployments, use this command for maximum optimization and security:

rncc HelloWorld.cs \
  --optimize=All \
  --security-analysis \
  --generate-artifacts=All \
  -o ./dist

Deployment

After compilation, you're ready to deploy your contract to the Neo blockchain.

1. Testing Locally

First, test your contract using Neo Express (local blockchain):

# Install Neo Express (if not already installed)
dotnet tool install Neo.Express -g

# Create a local blockchain
neoxp create

# Deploy contract
neoxp contract deploy HelloWorld.nef genesis

2. TestNet Deployment

Deploy to Neo TestNet for public testing:

# Using neo-cli
neo-cli
neo> open wallet your-wallet.json
neo> deploy HelloWorld.nef HelloWorld.manifest.json

3. WebGUI Deployment

If you generated a WebGUI, deploy it to R3E hosting:

# Generate and deploy WebGUI
rncc HelloWorld.cs \
  --generate-webgui \
  --deploy-webgui \
  --contract-address=0x123... \
  --network=testnet \
  --deployer-address=NiHYtfX...
