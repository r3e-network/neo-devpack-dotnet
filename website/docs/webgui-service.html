WebGUI Service Documentation

The R3E WebGUI Service automatically generates and hosts interactive web interfaces for your Neo smart contracts, making them accessible to non-technical users through a simple web browser.

Automatically creates UI from contract ABI

Wallet Integration

Support for NeoLine, O3, and WalletConnect

Responsive Design

Works perfectly on desktop and mobile

Real-time Updates

Live transaction status and event monitoring

Generating WebGUI

Generate a WebGUI for your contract using the RNCC compiler:

Basic Generation

# Generate WebGUI files locally
rncc MyContract.cs --generate-webgui

# Output files:
# bin/sc/MyContract.WebGUI/
#   ├── index.html        # Main application
#   ├── app.js           # Application logic
#   ├── style.css        # Styling
#   ├── manifest.json    # WebGUI configuration
#   └── assets/          # Icons and images

WebGUI Manifest

The generated manifest.json contains metadata about your contract interface:

{
  "name": "MyContract",
  "displayName": "My Smart Contract",
  "description": "Interactive interface for MyContract",
  "version": "1.0.0",
  "contract": {
    "address": "0x...",
    "network": "testnet",
    "abi": { ... }
  },
  "theme": {
    "primaryColor": "#00e599",
    "secondaryColor": "#00d4ff",
    "darkMode": true
  },
  "features": {
    "wallet": true,
    "events": true,
    "storage": true
  }
}

Deployment Options

Deploy your WebGUI using one of these methods:

Deploy to R3E Hosting Service

The easiest way to deploy your WebGUI:

# Generate and deploy in one command
rncc MyContract.cs \
  --generate-webgui \
  --deploy-webgui \
  --contract-address=0x1234567890abcdef... \
  --network=testnet \
  --deployer-address=NPvKVTGZapmFWABLsyvfreuqn73jCjJtN9

# Your WebGUI will be available at:
# https://service.neoservicelayer.com/contracts/0x1234567890abcdef...

R3E Hosting Benefits

  • Automatic SSL certificates
  • Global CDN distribution
  • Built-in analytics
  • No configuration required
  • Free tier available

Deploy to Static Hosting

Host on any static file hosting service:

Netlify

# Install Netlify CLI
npm install -g netlify-cli

# Deploy
cd bin/sc/MyContract.WebGUI
netlify deploy --prod

GitHub Pages

# Push to gh-pages branch
git subtree push --prefix=bin/sc/MyContract.WebGUI origin gh-pages

Vercel

# Install Vercel CLI
npm install -g vercel

# Deploy
cd bin/sc/MyContract.WebGUI
vercel --prod

Deploy to IPFS

Decentralized hosting using IPFS:

# Add to IPFS
ipfs add -r bin/sc/MyContract.WebGUI

# Pin the content
ipfs pin add QmYourHashHere

# Access via gateway
# https://ipfs.io/ipfs/QmYourHashHere

Or use a pinning service like Pinata:

# Using Pinata CLI
pinata-cli pin bin/sc/MyContract.WebGUI

Custom Components

Add custom UI components by modifying the generated files:

// app.js - Add custom validation
function validateInput(methodName, params) {
  if (methodName === 'transfer') {
    if (params.amount <= 0) {
      throw new Error('Amount must be positive');
    }
    if (!isValidAddress(params.to)) {
      throw new Error('Invalid recipient address');
    }
  }
  return true;
}

Self-Hosting WebGUI Service

Run your own instance of the R3E WebGUI hosting service:

Using Docker

# Clone the repository
git clone https://github.com/r3e-network/r3e-devpack-dotnet.git
cd r3e-devpack-dotnet/src/R3E.WebGUI.Service

# Configure environment
cp .env.example .env
# Edit .env with your settings

# Run with Docker Compose
docker-compose -f docker-compose.production.yml up -d

Configuration

Key environment variables for self-hosting:

# Database
DB_PASSWORD=YourSecurePassword
DOMAIN=your-domain.com
ADMIN_EMAIL=admin@your-domain.com

# Service Configuration
R3EWebGUI__BaseDomain=your-domain.com
R3EWebGUI__StorageBasePath=/app/webgui-storage
R3EWebGUI__MaxFileSizeKB=5120
R3EWebGUI__AllowedNetworks=["testnet", "mainnet"]

# Rate Limiting
R3EWebGUI__RateLimiting__EnableRateLimiting=true
R3EWebGUI__RateLimiting__PermitLimit=100
R3EWebGUI__RateLimiting__WindowMinutes=1

# Neo RPC Endpoints
NEO_RPC_TESTNET=https://test1.neo.coz.io:443
NEO_RPC_MAINNET=https://mainnet1.neo.coz.io:443

Architecture

Service Components

  • ASP.NET Core API: Handles WebGUI uploads and serving
  • SQL Server: Stores metadata and configurations
  • Nginx: Reverse proxy with SSL termination
  • File Storage: Stores WebGUI files
  • Let's Encrypt: Automatic SSL certificates

API Integration

Integrate with the WebGUI Service API for advanced use cases:

Upload WebGUI

# Upload WebGUI via API
curl -X POST https://api.r3e-gui.com/api/webgui/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: multipart/form-data" \
  -F "files=@webgui.zip" \
  -F "contractAddress=0x1234..." \
  -F "network=testnet"

API Endpoints

Available Endpoints

POST /api/webgui/upload Upload new WebGUI
GET /api/webgui/{address} Get WebGUI metadata
PUT /api/webgui/{address} Update existing WebGUI
DELETE /api/webgui/{address} Remove WebGUI
GET /api/stats/{address} Get usage statistics

JavaScript SDK

// Using the R3E WebGUI SDK
import { WebGUIClient } from '@r3e/webgui-sdk';

const client = new WebGUIClient({
  apiKey: 'YOUR_API_KEY',
  endpoint: 'https://api.r3e-gui.com'
});

// Upload WebGUI
const result = await client.upload({
  files: webguiFiles,
  contractAddress: '0x1234...',
  network: 'testnet',
  metadata: {
    name: 'My Contract',
    description: 'My awesome contract'
  }
});

console.log('WebGUI URL:', result.url);
